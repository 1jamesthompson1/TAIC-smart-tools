# Underlying Database

The smart tools are powered by a database of transport accident investigation reports from TAIC, ATSB, and TSB. This database is generated by the [TAIC-engine](https://github.com/1jamesthompson1/TAIC-engine) pipeline.

## How it works

The pipeline has four main stages:

1. **Scraping** — Report metadata, PDFs, safety issues, and recommendations are scraped from each agency's public website.

2. **Parsing** — Downloaded PDF reports are converted to plain text using PyMuPDF.

3. **AI Extraction** — An LLM reads each report and extracts structured information: safety issues, recommendations, and occurrence metadata (date, location, fatalities, injuries). Transport mode is known from the report ID and agency data.

4. **Vector Embedding** — All documents are embedded using Azure OpenAI and stored in a **LanceDB** vector database for semantic search.

## What's in the database

| Data type | Source |
|---|---|
| Report text & sections | Parsed from PDFs |
| Safety issues | Scraped from ATSB website; AI-extracted for TAIC, TSB, and pre-2008 ATSB |
| Recommendations | Scraped from TAIC and TSB websites; AI-extracted for ATSB |
| Summaries | Scraped from TAIC and ATSB report pages (not available for TSB) |
| Occurrence metadata | AI-extracted from report text |

Each document in the vector database has metadata including agency, year, transport mode, document type, location, fatalities, injuries, and a link to the original report.

## More information

For full details on the pipeline, extraction models, and how to run it yourself, see the [TAIC-engine GitHub repository](https://github.com/1jamesthompson1/TAIC-engine).
